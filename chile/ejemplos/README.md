## Estructura canónica

Cada ejemplo sigue el patrón:

1. **Frontmatter**: ejemplo, rama, nivel, archivos invocados, estado_revision.
2. **Escenario**: consulta realista del cliente (en cita).
3. **Respuesta del sistema con corpus cargado**: análisis paso a paso.
4. **Régimen aplicable**: normas + cuerpo legal.
5. **Procedimiento + pasos operativos**.
6. **Plazos críticos**.
7. **Cálculos / exposición** (cuando aplica).
8. **Red flags activadas**.
9. **Disclaimers**.
10. **Normas + skills invocados**.

## Niveles de dificultad

- **Básico**: una sola norma aplicable, plazo claro, sin cruces complejos.
- **Intermedio**: 2-4 normas + razonamiento mixto + decisión estratégica.
- **Avanzado**: 5+ normas, cruces multidisciplinarios, decisión de
  riesgo, monto significativo.

## Pendientes Fase 3

- Ejemplos por las 6 ramas restantes: civil, penal, familia,
  administrativo, privacidad, concursal. Mínimo 1 por rama, idealmente
  2-3.
- **Casos de prueba** integrados (`casos-prueba-diagnostico.md`) para
  validar que el sistema activa el perfil correcto.

## Cómo contribuir un ejemplo

1. Identificar caso típico real (sin datos identificables del cliente).
2. Aplicar el patrón canónico arriba.
3. Verificar normas + plazos contra el corpus actual.
4. Marcar estado_revision como `borrador-no-validado`.
5. PR.
